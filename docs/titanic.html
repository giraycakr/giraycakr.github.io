🚢 Titanic - Machine Learning from Disaster¶

1. Introduction¶

The sinking of the RMS Titanic on April 15, 1912 is one of the most infamous shipwrecks in history. Out of 2224 passengers and crew, 1502 lost their lives. While survival involved some luck, certain groups — women, children, upper-class passengers — had significantly higher survival rates. In this notebook, we'll build a machine learning model that predicts whether a passenger survived based on features like age, gender, ticket class, and family size. This is the classic Kaggle Getting Started competition, and our goal is to maximize prediction accuracy on the test set. Our approach:

Explore and visualize the data to understand survival patterns Clean the data and engineer meaningful features Train and compare multiple models Tune the best model and generate a final submission

Let's dive in!

8. Conclusion¶

In this notebook, we built a machine learning model to predict Titanic survival using the classic Kaggle Getting Started competition. Key findings from EDA:

Gender was the strongest predictor — around 74% of women survived vs only 19% of men Passenger class mattered significantly — first class passengers had much higher survival rates Family size had an impact — solo travelers and very large families had lower survival rates

Our approach:

Cleaned missing values (median for Age/Fare, mode for Embarked, dropped Cabin) Engineered new features: FamilySize, IsAlone, and Title extracted from passenger names Compared four models: Logistic Regression, KNN, Random Forest, and XGBoost Tuned top 2 models using GridSearchCV with 5-fold cross-validation Selected XGBoost as our final model (learning_rate=0.1, max_depth=3, n_estimators=200)

Possible improvements:

More advanced feature engineering (Age bins, Fare bins, Deck from Cabin) Trying ensemble methods like stacking or voting classifiers More sophisticated missing value imputation (grouped by Pclass and Sex) Feature selection to remove less useful features
